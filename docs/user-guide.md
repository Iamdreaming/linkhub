## 登录系统

### 首次登录

1. 在浏览器中打开 LinkHub 地址（如：`http://localhost:3000`）
2. 输入管理员密码（在 `.env` 文件中配置的 `ADMIN_PASSWORD`）
3. 点击"登录"按钮

![登录界面](https://via.placeholder.com/600x400?text=Login+Page)

### 安全提示

- 🔒 首次部署后请立即修改默认密码
- 🔒 使用强密码（至少12位，包含大小写字母、数字、特殊字符）
- 🔒 不要在公共场合暴露密码
- 🔒 定期更换密码（建议每3个月）

### 忘记密码

如果忘记密码，需要通过服务器修改：

```bash
# 编辑环境变量文件
nano backend/.env

# 修改 ADMIN_PASSWORD 的值
ADMIN_PASSWORD=your-new-password

# 重启服务
pm2 restart linkhub  # 或 docker-compose restart
```

## 链接管理

### 创建新链接

1. 点击页面顶部的"添加链接"按钮
2. 填写链接信息：
   - **短链代码**：自定义的短链标识（如：wx001, service01）
   - **链接名称**：便于识别的名称（如：客服微信、推广链接）
   - **目标链接**：实际的微信链接地址
   - **描述信息**：可选，用于备注说明
3. 点击"确定"保存

![创建链接](https://via.placeholder.com/600x400?text=Create+Link)

#### 短链代码规则

- ✅ 长度：3-20个字符
- ✅ 允许：字母、数字、下划线
- ✅ 唯一性：不能与已有短链重复
- ✅ 建议：使用有意义的代码，便于记忆

**示例**：
```
wx001      - 客服微信1号
service    - 客服入口
promo2024  - 2024年推广活动
group_a    - A组微信群
```

### 查看链接列表

链接列表显示所有已创建的短链接，包含以下信息：

- **短链代码**：固定的短链标识
- **名称**：链接名称
- **目标链接**：当前指向的微信地址
- **访问次数**：累计扫码次数
- **状态**：启用/禁用
- **创建时间**：链接创建日期
- **操作按钮**：编辑、统计、二维码、删除

![链接列表](https://via.placeholder.com/800x500?text=Link+List)

### 编辑链接

1. 在链接列表中找到要编辑的链接
2. 点击"编辑"按钮
3. 修改需要更新的信息：
   - 链接名称
   - 目标链接（重要：可以随时更新微信地址）
   - 描述信息
   - 启用/禁用状态
4. 点击"保存"

**注意**：短链代码创建后不可修改，这保证了短链的稳定性。

### 更新目标链接

这是 LinkHub 的核心功能！当您的微信链接变化时：

1. 编辑对应的短链接
2. 更新"目标链接"字段为新的微信地址
3. 保存更改

✨ **短链接和二维码保持不变**，用户扫描后会跳转到新地址！

### 启用/禁用链接

**单个操作**：
- 在编辑界面切换"启用状态"开关

**批量操作**：
1. 勾选要操作的链接
2. 点击"批量操作"按钮
3. 选择"启用"或"禁用"

禁用的链接：
- ❌ 无法访问（返回410错误）
- ✅ 保留所有数据和统计
- ✅ 可以随时重新启用

### 删除链接

**警告**：删除操作不可恢复！

**单个删除**：
1. 点击链接的"删除"按钮
2. 确认删除操作

**批量删除**：
1. 勾选要删除的链接
2. 点击"批量操作" → "删除"
3. 确认删除操作

删除后：
- ❌ 短链接无法访问
- ❌ 所有访问统计数据被清除
- ❌ 无法恢复

### 搜索和筛选

**搜索功能**：
- 在搜索框输入关键词
- 支持搜索：链接名称、短链代码、描述

**筛选功能**：
- 按状态筛选：全部/启用/禁用
- 按访问量排序：从高到低/从低到高
- 按创建时间排序：最新/最早

## 二维码使用

### 查看二维码

1. 在链接列表中点击"二维码"按钮
2. 弹出窗口显示二维码图片
3. 可以直接扫描测试

![二维码预览](https://via.placeholder.com/400x400?text=QR+Code+Preview)

### 下载二维码

**单个下载**：
1. 点击二维码预览窗口的"下载"按钮
2. 选择格式：PNG（推荐）或 SVG
3. 选择尺寸：
   - 小：200x200px（适合网页）
   - 中：300x300px（推荐）
   - 大：500x500px（适合打印）
   - 超大：1000x1000px（高清打印）

**批量下载**：
1. 勾选多个链接
2. 点击"批量操作" → "下载二维码"
3. 系统生成ZIP压缩包
4. 下载并解压使用

### 二维码自定义

在下载时可以自定义：

- **尺寸**：100-1000像素
- **格式**：PNG（位图）或 SVG（矢量图）
- **边距**：0-10像素
- **颜色**：前景色和背景色（十六进制）

**使用建议**：
- 📱 手机扫描：300x300px PNG
- 🖨️ 打印海报：800x800px PNG
- 🎨 设计使用：SVG格式（可无损缩放）

### 二维码使用场景

1. **线上推广**
   - 网站嵌入
   - 社交媒体分享
   - 电子邮件营销

2. **线下推广**
   - 打印海报
   - 宣传单页
   - 名片背面
   - 展架展板

3. **多渠道分发**
   - 不同渠道使用不同短链
   - 统一的二维码样式
   - 便于追踪效果

1. 在链接列表中点击"统计"按钮

**概览数据**：

- **今日访问**：今天的访问次数
- **本周访问**：最近7天的访问次数
- **本月访问**：最近30天的访问次数
- **日均访问**：平均每天的访问次数

**访问趋势图**：

**时段分析**：
- 🕐 24小时访问分布
- 🕐 找出访问高峰时段

**来源分析**：
- 🔗 显示访问来源（Referer）
- 🔗 了解流量来源渠道
- 🔗 评估推广效果

**最近访问**：
- 📝 最新的访问记录

## 数据备份

### 自动备份

系统默认每天凌晨3点自动备份数据库。

**配置说明**：
```bash
BACKUP_ENABLED=true              # 启用自动备份
BACKUP_SCHEDULE=0 3 * * *       # 每天凌晨3点
BACKUP_RETENTION_DAYS=30        # 保留30天
```

### 手动备份

1. 进入"系统设置"页面
2. 点击"立即备份"按钮
3. 等待备份完成
4. 查看备份列表

### 查看备份

在"备份管理"页面可以看到：
- 备份文件名
- 备份时间
- 文件大小
- 操作按钮（下载/恢复/删除）

### 恢复备份

**警告**：恢复备份会覆盖当前数据！

1. 在备份列表中找到要恢复的备份
2. 点击"恢复"按钮
3. 确认操作
4. 等待恢复完成
5. 重启服务

### 下载备份

1. 点击备份的"下载"按钮
2. 保存到本地
3. 妥善保管备份文件

**建议**：
- 💾 定期下载备份到本地
- 💾 使用云存储保存重要备份
- 💾 保留多个时间点的备份

## 常见问题

### Q1: 短链接无法访问？

**可能原因**：
1. 链接已被禁用
2. 短链代码输入错误
3. 服务器未启动

**解决方法**：
- 检查链接状态是否为"启用"
- 确认短链代码拼写正确
- 检查服务器运行状态

### Q2: 二维码扫描后没反应？

**可能原因**：
1. 目标链接格式错误
2. 微信链接已失效
3. 网络连接问题

**解决方法**：
- 检查目标链接是否正确
- 测试微信链接是否有效
- 确认网络连接正常

### Q3: 统计数据不准确？

**可能原因**：
1. 浏览器缓存
2. 同一IP多次访问
3. 爬虫访问

**说明**：
- 系统记录所有访问，包括重复访问
- 可以通过IP地址识别独立访客
- 建议结合多个指标综合分析

### Q4: 忘记管理员密码？

**解决方法**：
```bash
# 通过服务器修改
nano backend/.env
# 修改 ADMIN_PASSWORD
# 重启服务
```

### Q5: 如何修改短链代码？

**答案**：短链代码创建后不可修改，这是为了保证链接的稳定性。

**替代方案**：
1. 创建新的短链接
2. 将旧链接的访问重定向到新链接
3. 逐步替换使用场景

### Q6: 数据库文件在哪里？

**位置**：
- Docker部署：`./data/linkhub.db`
- 传统部署：`backend/data/linkhub.db`

**备份位置**：
- `./data/backups/` 或 `backend/data/backups/`

## 最佳实践

### 链接命名规范

建议使用统一的命名规范：

```
用途-编号
例如：
wx001, wx002      - 客服微信
promo2024q1       - 2024年第一季度推广
group_tech        - 技术交流群
service_vip       - VIP客服
```

### 定期维护

**每周**：
- 检查链接状态
- 查看访问统计
- 更新失效链接

**每月**：
- 清理无用链接
- 分析访问趋势
- 优化推广策略

**每季度**：
- 下载数据备份
- 更新管理密码
- 检查系统更新

### 安全建议

1. **密码安全**
   - 使用强密码
   - 定期更换
   - 不要共享

2. **访问控制**
   - 使用HTTPS
   - 限制管理IP（可选）
   - 启用访问日志

3. **数据保护**
   - 启用自动备份
   - 定期下载备份
   - 测试恢复流程

### 推广技巧

1. **多渠道使用**
   - 为不同渠道创建不同短链
   - 便于追踪效果
   - 优化投放策略

2. **二维码设计**
   - 保持足够大小（至少3x3cm）
   - 确保对比度清晰
   - 添加引导文字

3. **数据分析**
   - 定期查看统计
   - 对比不同时期
   - 调整推广时间

### 性能优化

1. **定期清理**
   - 删除无用链接
   - 清理过期日志（90天前）
   - 优化数据库

2. **监控指标**
   - 关注响应时间
   - 检查错误日志
   - 监控磁盘空间

3. **扩展准备**
   - 访问量增长时考虑升级配置
   - 必要时使用CDN
   - 考虑数据库迁移
